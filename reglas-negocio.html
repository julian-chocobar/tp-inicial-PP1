Reglas de Negocio

Sistema de Control de Ingreso con Reconocimiento Facial

Documento que establece las reglas de negocio para el prototipo de control de acceso en una PYME alimenticia utilizando tecnología de reconocimiento facial.

🔐 Reglas de Control de Acceso y Seguridad

RN-001: Control de Acceso por Zonas

El sistema debe permitir definir niveles de acceso diferentes según las áreas de la planta (producción, almacén, oficinas).

RN-002: Horarios de Acceso

El acceso debe estar restringido según los turnos laborales establecidos para cada empleado.

RN-003: Registro de Eventos

Todo intento de acceso (exitoso o fallido) debe quedar registrado con timestamp y detalles del evento.

RN-004: Múltiples Métodos de Autenticación

El sistema debe permitir autenticación por reconocimiento facial + código PIN como método de respaldo.

👤 Reglas de Reconocimiento Facial

RN-101: Umbral de Confianza

El sistema debe requerir un mínimo de 95% de confianza en la identificación para permitir el acceso.

RN-102: Múltiples Registros Faciales

Cada empleado debe tener al menos 3 registros faciales desde diferentes ángulos y condiciones de iluminación.

RN-103: Actualización de Modelos

El sistema debe permitir la actualización periódica de los modelos faciales para mejorar la precisión.

RN-104: Detección de Suplantación

El sistema debe incluir mecanismos básicos de detección de suplantación (detección de vivacidad).

👥 Reglas de Gestión de Personal

RN-201: Roles y Permisos

Definición clara de roles (operario, supervisor, administrador) con permisos específicos para cada uno.

RN-202: Turnos Laborales

Gestión de turnos con horarios específicos y control de horas extras automático.

RN-203: Ausencias y Vacaciones

Integración con sistema de gestión de ausencias para bloquear acceso durante períodos no laborales.

RN-204: Notificaciones

Sistema de alertas para accesos fuera de horario o intentos fallidos repetidos.

⚖️ Reglas de Cumplimiento Normativo

RN-301: Protección de Datos

Cumplimiento con normativas de protección de datos personales (Ley 25.326 en Argentina).

RN-302: Seguridad Alimentaria

Control de acceso estricto a áreas de producción para cumplir con BPM (Buenas Prácticas de Manufactura).

RN-303: Auditoría

Mantenimiento de registros de acceso por 6 meses mínimo para auditorías internas y externas.

RN-304: Consentimiento

Obtención de consentimiento explícito para el uso de datos biométricos según normativa vigente.

💻 Reglas Técnicas y de Implementación

RN-401: Disponibilidad

El sistema debe tener una disponibilidad mínima del 99% durante horarios laborales.

RN-402: Respuesta

Tiempo de respuesta menor a 2 segundos para el reconocimiento facial.

RN-403: Backup

Sistema de respaldo automático de la base de datos y registros de acceso.

RN-404: Escalabilidad

Arquitectura que permita escalar hasta 100 usuarios concurrentes.
